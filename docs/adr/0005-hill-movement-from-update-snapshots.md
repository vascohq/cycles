# Hill movement comes from update snapshots, not a continuous log

We show how scopes move on the hill chart — trails, ghosts, the arrow-scrubbable history — by diffing the Hill Snapshot frozen on each needle update against the one before it (and, on the live map, the latest snapshot against the current dot positions). We deliberately do **not** record every drag as it happens. The only persisted record of hill movement is the per-update snapshot that already exists on each `PitchUpdate`.

We chose this over a continuous movement log because updates are meant to communicate confidence, not activity. Intra-week wiggle — a dot creeping forward Wednesday and sliding back Friday — is activity noise; the signal we care about is "between two checkpoints, did this scope resolve, stall, or regress?" A continuous log would invite people to read the squiggles, which quietly undermines the deliberate Tuesday-update cadence the whole product is built around. It also keeps the feature free of new persistence: the snapshot is already captured, so trails, labels, and history are all pure functions over data we already store. The cost is that we can never reconstruct sub-update motion — accepted on purpose.

Two consequences worth recording. First, the Hill Snapshot is enriched to freeze each scope's `title` and `tier` (not just `scopeId` and `hill_progress`), so a scope deleted since the snapshot still renders as a *named* ghost rather than an anonymous dot; that ghost self-expires once the next update is posted and the scope falls out of the baseline. Second, stagnation is made visible (a dot with no trail stands out against neighbours that have one) but is **never** auto-judged — no system-computed "at risk" flag. That keeps faith with [ADR 0004](0004-needle-position-independent-and-manual.md): the chart surfaces the question "why isn't this moving?", and the team answers it by moving the needle themselves. Per-scope labels like "didn't move" or "over the hill" are allowed because they describe the delta; they do not render a verdict.
